初探MongoDB

2017-03-01

MongoDB,作为一种NoSQL的代表数据库，最近变得越来越热门。跟师兄们聊天时发现，我原来对NoSQL以及MongoDB的了解尚浅。
NoSQL，指的是非关系型的数据库。NoSQL有时也称作Not Only SQL的缩写，是对不同于传统的关系型数据库的数据库管理系统的统称。
NoSQL用于超大规模数据的存储。（例如谷歌或Facebook每天为他们的用户收集万亿比特的数据）。这些类型的数据存储不需要固定的模式，无需多余操作就可以横向扩展。代表着不仅仅是SQL,没有声明性查询语言；没有预定义的模式；键 - 值对存储，列存储，文档存储，图形数据库;最终一致性，而非ACID属性;非结构化和不可预知的数据。
NoSQL满足CAP定理 :一致性(Consistency) (所有节点在同一时间具有相同的数据),可用性(Availability) (保证每个请求不管成功或者失败都有响应),分隔容忍(Partition tolerance) (系统中任意信息的丢失或失败不会影响系统的继续运作),一个分布式系统不可能同时很好的满足一致性，可用性和分区容错性这三个需求，最多只能同时较好的满足两个。高性能，高可用性和可伸缩性。SQL满足ACID,原则，而NoSQL满足BASE原则。
ACID: 原子性(Atomicity),一致性(Consistency),隔离性(Isolation),持久性 (Durable)
BASE: 基本可用(Basically Available),软状态/柔性事务(Soft state),最终一致性 (Eventual consistency)
MongoDB进行数据管理时，可以通过建立Key值索引来索引和管理文件。如果文档的索引字段值超过了索引键的限制，MongoDB不会将任何文档转换成索引的集合。这就导致了集合中索引不能超过64个，索引名的长度不能超过125个字符，一个复合索引最多可以有31个字段。
用MongoDB存储与管理空间数据，是现在很多公司常用的办法。空间数据可以用geoJson来存储，MongoDB对Json类型数据的存储和管理有很大的优势，只是，如何进行传统的空间范围查询或者空间近邻属性查询呢？有人用过B树来做索引查询，也有人在做用R树索引的研究，希望我今晚听完师兄研究的东西会有所感悟吧。

在网上下载了很多论文的相关信息，一篇论文会提供他的通讯地址。如:”Daniel, M.-P., Groupe Cognition Humaine, LIMSI-CNRS (UPR 3251), Université de Paris-Sud, BP 133, 91403 Orsay Cedex, France; “ ，怎么获取其中的 国家信息呢？
1.首先我在网上下载了世界各国家的英文单词。并把他们添加到了一个.properties文件中。想通过正则表达式进行匹配和抓取，代码如下：

2.JAVA通过.properties文件的读取来实现Enumeration的实例化。这样可以自动建立正则表达式。但是文件运行的时候出现了问题。由于很多国家是中间有空格的，存储properties文件的时候我一开始是直接复制粘贴，其中也必然带有空格。而在properties文件中，会自动把空格看成key=value的=号，读取时只能读取空格前的key，这样会造成一个问题就是united states进行正则匹配时只能匹配到united.

3.通过各种查找，我终于找到了解决的办法。就是用转义表示空格。网上有很多种说法，一种是说在空格前加反斜杠如\ ，还有一种是说英文空格\u0020，中文空格是\u3000.经亲测，证实：如果想过滤 的是英文词组，不止是一个英文单词，请用\u0200代替中间的空格存储在properties文件中，就能正常的进行国家匹配和抓取了。

利用Hexo搭建个人博客

2017-02-16

一直以来都想有一个自己的博客，开学之前，终于把自己的博客搭好了。
搭建博客在网上一搜有很多写的很好的教程，我主要是把我这个0基础小白遇到的问题记录下来，希望能帮到跟我一样有过困扰的宝宝们。
我主要参考的是WX_JIN 和代码咖啡两个的博客，链接如下：

  1. http://blog.csdn.net/wx_jin/article/details/51027783
  2. http://www.jianshu.com/p/e99ed60390a8
    使用的是Yilia的HEXO主题。
    搭建的大概流程：
    1.搭建的第一步是：node.js、 git 、hexo的下载安装。这几步中可能遇到的问题是由于限网的原因，可能进不了官网下载相应的资源，
    我的解决办法是：1）用lattern翻墙试试下载安装 2）有百度离线下载工具打开链接并下载。
    2.初始化hexo后会在指定的文件夹生成hexo文件夹。这个文件夹中最重要的就是config.yml文件。我们需要修改这个配置文件才能将hexo的效果应用到我们想绑定的页面上。在这个过程中可能会遇到hexo -g 等命令行出现command not found 的情况。此时原因可能是虽然修改了配文件，但是运行代码的的windows终端不会自动更新，此时我们需要关掉终端，并重新在hexo文件夹点击右键如图：
    出现的终端的正常显示是像这样的。
    3.对下载的主题进行修改，可能会遇到以下的问题：1）不知道在哪里修改自己的头像，怎么自定义自己的皮肤。出现此情况的原因是因为hexo文件夹下有config.yml文件，theme 文件夹里面的主题文件夹中也有config.yml文件。修改头像是在theme文件夹中的config文件修改。头像的方式有两种，一种是存储在theme的source文件夹，设置相对路径，一种是使用图床软件，把自己的照片上传，获取自己照片的URL。2）修改了以后上传到github时，与域名绑定又会失效。 这是因为hexo -deploy会替换掉github中的所有文件，包括CNAME 文件，解决的办法是在hexo 的source 文件重新生成CNAME文件，这样每次hexogenerate的时候就会重新生成一个CNAME文件，每次博客的更新都可以更新到自己的域名和github主页上。
